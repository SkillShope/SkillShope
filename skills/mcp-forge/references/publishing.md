# Testing & Publishing

After generating your MCP server, test it locally and publish it.

## Local Testing

```bash
cd <server-name>
npm install
npm run build

# Test with MCP inspector
npx @modelcontextprotocol/inspector node dist/index.js
```

The MCP Inspector opens a web UI where you can:
- See all registered tools
- Call tools with test parameters
- Inspect responses and errors

## Testing with Claude Code

Add the server to your Claude Code config:

```json
{
  "mcpServers": {
    "my-api": {
      "command": "node",
      "args": ["path/to/<server-name>/dist/index.js"],
      "env": {
        "MCP_API_KEY": "your-test-key"
      }
    }
  }
}
```

Then ask Claude Code to use the tools:
- "List all customers"
- "Create a new charge for $50"
- "Get payment intent pi_xxx"

## Publishing to npm

```bash
cd <server-name>

# Update package.json name and version
npm version 1.0.0

# Publish (requires npm account)
npm publish --access public
```

Users install with:
```bash
npx @yourorg/server-name
```

## Publishing to GitHub

```bash
cd <server-name>
git init
git add -A
git commit -m "Initial MCP server generated by MCP Forge"
gh repo create yourname/server-name --public --push
```

Users install with:
```bash
npx github:yourname/server-name
```

## Listing on Skill Shope

After publishing to npm or GitHub:

1. Go to [skillshope.com/publish](https://skillshope.com/publish)
2. Select **MCP Server** as the type
3. Set the source URL to your npm package or GitHub repo
4. Add the install command: `npx @yourorg/server-name`
5. Tag it with the API name and relevant keywords

## Traps to Avoid

- **Don't hardcode API keys** — Always use environment variables. Generated servers enforce this.
- **Don't publish with test credentials** — Check your `.env` is in `.gitignore` before pushing.
- **Don't skip local testing** — A broken MCP server gets bad reviews fast.
- **Don't generate from incomplete specs** — Missing schemas = missing types = runtime errors. Validate the spec first.
